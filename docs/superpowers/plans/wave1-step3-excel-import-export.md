# Wave-1 Step 3 — Excel Import/Export for Assets

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the user-mandated client-side Excel export and import for the assets table on `/assets`. Export reuses the current filter state. Import is a 3-step modal: upload + template, dry-run preview with green/yellow/red row classification, commit batch. Create-only (no edit-via-import). Mandatory in Phase 1, NOT Phase 2.

**Architecture:** Pure-JS service layer (`src/lib/excel/*`) handles workbook ↔ rows mapping with no React or Firestore imports. `xlsx` (SheetJS) for parsing/writing. Export is a one-shot click that calls the service with the already-filtered list. Import is a `<AssetImportDialog>` component that drives validation against current Firestore state (categories, statuses, branches, employees) and commits via the existing `firestoreAssetRepository.create` — same transactional inventory-code path as the form, so audit logs and counter increments are covered for free. No new Firestore rules. No Cloud Function. No new collections.

**Tech Stack:** `xlsx@^0.18.5` (SheetJS), React 19, react-i18next, existing shadcn/ui Dialog/Button/Alert/Spinner. Vitest + RTL. Firebase mocks per `src/test/setup.js`.

- Category / branch / employee creation on the fly.
- Server-side Cloud Function for very large files. Threshold sanity-check: spec calls out "huge files (>10k rows)" as Phase-2; we keep the Step-3 cap at 5,000 rows client-side and surface a UX-level error at the Step-2 preview if exceeded.
- Localized template column headers (we use ENGLISH header strings as canonical machine-friendly identifiers; the template includes a hidden second header row with the user-locale label for human reference). Keeping ENGLISH as the canonical column key means templates round-trip across locales — this is the deliberate choice.

## Decisions baked in (not asking the user)

| Question | Decision | Source |
|---|---|---|
| Excel library | `xlsx@^0.18.5` (SheetJS Community) | Industry standard, MIT-licensed, browser+node, single dep |
| Header row 1 = canonical English keys | Yes — keys are stable identifiers | round-trip across locales |
| Header row 2 = friendly localized label | Yes — info-only, ignored on parse | UX |
| Date columns | ISO yyyy-mm-dd OR Excel serial number | xlsx supports both natively |
| Currency / price column | Plain number; we don't carry currency in Step 2/3 — `purchasePrice` is a unitless number | matches Step 2 schema |
| `assignedTo.kind` column values | lowercase: `warehouse` / `employee` / `branch` / `department` | matches `ASSIGNMENT_KINDS` constants |
| `assignedTo.id` column shape | string id (asset_admin can find the id on existing list pages) | matches storage |
| Holder name column | info-only on import, populated on export for human reference | directive |
| Multi-lang category Name | Three columns: `Name (RU)`, `Name (EN)`, `Name (HY)` | preserves storage shape |
| Single-lang category Name | User fills `Name (RU)` ; on import we mirror RU into EN+HY (matches `sanitizeCategoryInput` mirror behavior in `domain/categories.js`) | round-trip safe |
| Status column on import | string id (e.g. `warehouse`, `assigned`); falls back to default `warehouse` with yellow warning if blank | matches `DEFAULT_ASSET_STATUS_CODE` |
| Inventory code on import | Generated by repo (`createAsset`) — never read from file | counter authoritative |
| Inventory code on export | Read straight from `asset.inventoryCode` | matches storage |
| Date in export | ISO `yyyy-mm-dd` string from `Timestamp.toDate()` | unambiguous |
| Filename | `assets_export_YYYY-MM-DD.xlsx` (UTC date) | directive |
| Template filename | `assets_import_template.xlsx` | directive |
| Failure-report filename | `assets_import_failures_YYYY-MM-DD.xlsx` | symmetric |
| Row cap (client-side) | 5000 — refuse with red banner over that | UX safety |
| Empty file | Red banner "Файл не содержит строк" on Step 2 | UX |
| Wrong header order in upload | Tolerate any column order; missing columns produce red rows for affected fields, not a global abort | UX |
| Department column | Accept `assignedTo.kind=department` + raw id verbatim — no resolution because departments collection isn't built yet (Step 2 plan §"Decisions baked in"). On import this passes through as a string id. On export the holder cell renders the raw id. | matches Step 2 |
| Roles | Same as Asset add: `super_admin` and `asset_admin`. `tech_admin` sees no buttons. | matches Step 2 plan |
| firestore.rules / storage.rules | NOT changed | Step 3 doesn't add new collections or fields |
| Audit per imported row | The existing `createAsset` writes one `audit_logs` row per asset (action `create`); no extra audit row per import is added | reuse |

## Validation rules (preview pass) — green/yellow/red

Per row, in this order. The first matched red short-circuits to red; multiple yellows accumulate.

1. **Required category** — categoryId resolves OR categoryName resolves to exactly one active category. Missing/ambiguous/inactive → red `errorImportCategoryRequired`.
2. **Required name** — for the resolved category:
   - if `requiresMultilang === true`: at least one of `nameRu` / `nameEn` / `nameHy` non-empty. All three empty → red `errorImportNameRequired`. Partial fill → yellow `warnImportNamePartialLocales` and the importer mirrors the first non-empty across the empties (matches `sanitizeAssetInput`-without-multilang behavior — but only for multi-lang categories we use the partial mirror to avoid losing data; however the standard sanitize produces a locale map with empties, which `firestore.rules.isValidAssetName` accepts since each locale field is `is string`. So partial is technically valid; we still surface the warning to alert the user).
   - if `requiresMultilang === false`: `nameRu` required. Missing → red `errorImportNameRequired`. The importer copies `nameRu` to plain string (matching Step-2 storage shape for single-lang).
3. **Required `assignedToKind`** ∈ `{warehouse,employee,branch,department}`. Missing/unknown → red `errorImportAssignedKindRequired`.
4. **Warehouse rule** — if kind=warehouse and `assignedToId` present → yellow `warnImportWarehouseIdIgnored`; importer drops the id (matches `sanitizeAssetInput`).
5. **Holder existence**:
   - kind=employee: `assignedToId` must resolve to active employee. Missing/inactive/unknown id → red `errorImportEmployeeUnknown`.
   - kind=branch: `assignedToId` must resolve to active branch. Missing/inactive/unknown → red `errorImportBranchUnknown`.
   - kind=department: `assignedToId` must be a non-empty string (no resolution because departments collection isn't built; see Step-2 plan). Empty → red `errorImportDepartmentRequired`.
6. **Brand / model / serialNumber** must be ASCII-only when present (mirror domain rule). Cyrillic / non-ASCII → red `errorImportAsciiOnly` with `field` flag.
7. **Status** — `statusId` if present must resolve to an active status; otherwise → yellow `warnImportStatusFallback`, importer falls back to `DEFAULT_ASSET_STATUS_CODE` (`'warehouse'`).
8. **Status × kind compatibility**:
   - kind=warehouse → status must have `isAssignable === false`.
   - kind ∈ {employee,branch,department} → status must have `isAssignable === true`.
   - mismatch → red `errorImportStatusKindMismatch`.
9. **branchId requirement**:
   - kind ∈ {warehouse,branch}: `branchId` required & must resolve. Missing/unknown → red `errorImportBranchIdRequired`.
   - kind ∈ {employee,department}: `branchId` if present is dropped (matches sanitize); silent (no warning).
10. **purchaseDate** if present: ISO yyyy-mm-dd OR Excel serial number → parses to a valid Date. Invalid → red `errorImportPurchaseDate`.
11. **purchasePrice** if present: parses to a finite number ≥ 0. Invalid → red `errorImportPurchasePrice`.
12. **Inventory-code conflict** — if a non-empty `inventoryCode` cell exists AND it matches an existing asset's inventoryCode → red `errorImportInventoryCodeConflict`. (Free-form imports leave it empty; this rule fires only when the user uploaded a re-export by mistake.)

Aggregation:
- `red` = any red rule fired.
- `yellow` = no red, ≥ 1 yellow.
- `green` = no red, no yellow.

Output `validateRow` shape:

```js
/** @typedef {{
 *   status: 'green'|'yellow'|'red',
 *   errors: Array<{ rule: string, field?: string }>,    // i18n keys
 *   warnings: Array<{ rule: string, field?: string }>,
 *   normalized: import('@/domain/assets.js').AssetInput | null   // null when red
 * }} ValidationResult */
```

### `src/lib/excel/assetExportService.js` (≤ 200 LOC)

Pure functions, no React, no Firestore SDK:

```js
export function rowsToWorkbook(assets, ctx) { ... }
// ctx = { categoriesById, statusesById, branchesById, employeesById, locale }
// Builds the AOA, returns an `xlsx` workbook object.

export function workbookToBlob(wb) { ... }
// `xlsx.write(wb, { bookType: 'xlsx', type: 'array' })` → Blob with the
// XLSX MIME type.

export function downloadFilename(date = new Date()) { ... }
// "assets_export_YYYY-MM-DD.xlsx" (UTC).

export function buildExportSheetName() { return 'Assets'; }
```

`rowsToWorkbook` algorithm:

1. Build header row 1 from `COLUMN_KEYS` (machine-readable keys).
2. Build header row 2 from `t(headerLabelKey, lng=ctx.locale)` for each key (info-only, helps users read).
3. For each asset, emit one row by reading from the four resolution maps (categoriesById, statusesById, branchesById, employeesById).
   - `nameRu/nameEn/nameHy`: when `asset.name` is a string (single-lang), put the same value in all three columns. When it's a map, use each locale's value.
   - `holderName`: localized resolution; for departments fall back to raw id.
   - `purchaseDate`: `asset.purchaseDate?.toDate?.()?.toISOString().slice(0,10) ?? ''`.
   - `createdAt`: same shape.
4. Use `XLSX.utils.aoa_to_sheet(rows)` then `XLSX.utils.book_append_sheet(wb, ws, 'Assets')`.

Pure functions, no React, no Firestore SDK:

```js
export function workbookToRows(wb) { ... }
// Assumes "Assets" sheet (case-insensitive) or first sheet. Returns:
//   { headerKeys: string[], rows: Array<Record<string, string|number|null>>, errors: string[] }
// Tolerates extra columns. Warns on missing required headers via errors[].

export function validateRow(rawRow, rowIndex, ctx) { ... }
// ctx = {
//   categories: Category[], statuses: AssetStatus[], branches: Branch[],
//   employees: Employee[], existingInventoryCodes: Set<string>
// }
// Returns ValidationResult.

export function buildFailureReport(failures) { ... }
// failures = Array<{ rowIndex: number, errors: ..., raw: Record<...> }>
// Returns a workbook with one sheet listing each failure + reason text
// (i18n keys joined with '; ').
```

`validateRow` does NOT touch Firestore directly — `ctx` is the in-memory snapshot of the catalogs. The component preloads them via the existing hooks.

`workbookToRows` reads the first sheet (whatever name) and assumes:
- Header row 1 = column keys.
- Header row 2 = labels (skipped — we ignore it).
- Data starts at row 3.

We detect "row 2 is labels" by checking if **all** values in row 2 are strings AND none of them match a known column key — in that case treat row 2 as the label row. Otherwise data starts at row 2 (so a hand-rolled file with only one header row still works).

## UI plan

### `AssetExportButton.jsx` (≤ 90 LOC)

Props: `{ assets, categoriesById, statusesById, branchesById, employeesById, disabled }`.

```jsx
export default function AssetExportButton({ assets, categoriesById, statusesById, branchesById, employeesById, disabled }) {
  const { i18n, t } = useTranslation('assets');
  function handleExport() {
    const wb = rowsToWorkbook(assets, {
      categoriesById, statusesById, branchesById, employeesById,
      locale: i18n.resolvedLanguage ?? 'ru',
    });
    const blob = workbookToBlob(wb);
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = downloadFilename();
    document.body.appendChild(a);
    a.click();
    a.remove();
    URL.revokeObjectURL(url);
  }
  return (
    <Button variant="ghost" size="sm" className="gap-2" onClick={handleExport} disabled={disabled || assets.length === 0}>
      <Download className="h-4 w-4" aria-hidden="true" />
      {t('export')}
    </Button>
  );
}
```

The button is wired in the page to receive the *already-filtered* `rows` (NOT the unfiltered `assets`) — this matches the directive "Reuse the current filter state".

### `AssetImportDialog.jsx` (≤ 500 LOC)

Three internal stages: `'upload' | 'preview' | 'commit' | 'done'` + `error`.

State:
```js
const [stage, setStage] = useState('upload');
const [file, setFile] = useState(null);
const [rows, setRows] = useState([]);            // raw rows from parser
const [results, setResults] = useState([]);      // validateRow results
const [progress, setProgress] = useState({ done: 0, total: 0 });
const [outcome, setOutcome] = useState(null);    // { created, failed, failureWorkbookBlob }
const [readError, setReadError] = useState(null);
```

Hooks:
- `useCategories`, `useAssetStatuses`, `useBranches`, `useEmployees`, `useAssets` (the last one is needed to build `existingInventoryCodes` for the conflict rule). All four catalog hooks already exist.

Stage 1 (`upload`):
- Title: t('importDialogTitle')
- Description paragraph.
- "Скачать шаблон" link → builds an empty workbook on the fly (no live data) and downloads. Implemented inline as a function `downloadTemplate(t)` that uses `assetExportService.rowsToWorkbook([], ctx)` with empty maps.
- File picker `<input type="file" accept=".xlsx,.xls" />`. On change:
  1. Read as ArrayBuffer.
  2. Call `workbookFromArrayBuffer`.
  3. Call `workbookToRows`.
  4. Cap check: rows.length > 5000 → `readError = errorImportTooManyRows`, stay on `upload`.
  5. Run `validateRow` for each row using the loaded ctx (catalogs + Set of existing inventory codes from `useAssets`).
  6. Move to `preview`.

Stage 2 (`preview`):
- Header counts: `${greens} green, ${yellows} yellow, ${reds} red`.
- Filter pills (All / Green / Yellow / Red).
- Table: row index, status dot (green/yellow/red), inventoryCode preview "—" (server allocates), category name (resolved), name (RU shown), holder kind+name, status. Failure-cell rows show the comma-joined `t(error.rule)` strings under each row.
- "Назад" (back to upload), "Импортировать" button. Disabled while `reds > 0`.

Stage 3 (`commit`):
- Full-screen modal content with progress bar.
- Iterates over `results.filter(r => r.status !== 'red')` and calls
  `firestoreAssetRepository.create(r.normalized, actor, { category })`
  one at a time (NOT parallel — counter is transactional and parallel callers
  would still serialize but at the cost of more retries). After each call
  bumps `progress.done`. Catches per-row errors:

  ```js
  try {
    await repo.create(input, actor, { category });
    successes++;
  } catch (err) {
    failures.push({ rowIndex: r.rowIndex, raw: r.raw, errors: [{ rule: 'errorImportRepositoryFailed' }] });
  }
  ```
- After loop, `outcome = { created: successes, failed: failures.length, failureWorkbookBlob: failures.length ? buildFailureReport(failures, blobFromBuilder) : null }`.
- Move to `done`.

Stage 4 (`done`):
- Success banner with `Импортировано: N успешно, M пропущено`.
- If `M > 0`: download link for failure report.
- "Закрыть" button → `onClose()`.

```js
/**
 * Pure-JS Excel export service for the assets list.
 *
 * No React, no Firestore, no I/O beyond the SheetJS encode call.
 * `rowsToWorkbook` is the deterministic entry point — given the same input
 * it always produces the same workbook — which keeps the tests predictable.
 */

import * as XLSX from 'xlsx';

import { COLUMN_KEYS, COLUMN_LABEL_KEYS } from '@/lib/excel/columns.js';
import { localize } from '@/lib/localize.js';
import { formatEmployeeName } from '@/domain/employees.js';
import { ASSIGNMENT_KINDS } from '@/domain/assets.js';

function pad2(n) { return String(n).padStart(2, '0'); }

function isoDateUTC(d) {
  if (!d) return '';
  if (typeof d.toDate === 'function') d = d.toDate();
  if (!(d instanceof Date) || Number.isNaN(d.valueOf())) return '';
  return `${d.getUTCFullYear()}-${pad2(d.getUTCMonth() + 1)}-${pad2(d.getUTCDate())}`;
}

function nameForLocale(asset, locale) {
  const v = asset?.name;
  if (v == null) return { ru: '', en: '', hy: '' };
  if (typeof v === 'string') return { ru: v, en: v, hy: v };
  if (typeof v === 'object') {
    return {
      ru: v.ru ?? localize(v, locale) ?? '',
      en: v.en ?? localize(v, locale) ?? '',
      hy: v.hy ?? localize(v, locale) ?? '',
    };
  }
  return { ru: '', en: '', hy: '' };
}

function holderName(asset, ctx) {
  const k = asset?.assignedTo?.kind;
  if (k === ASSIGNMENT_KINDS.WAREHOUSE) {
    const b = asset.branchId ? ctx.branchesById.get(asset.branchId) : null;
    return b ? localize(b.name, ctx.locale) : '';
  }
  if (k === ASSIGNMENT_KINDS.BRANCH) {
    const b = asset.branchId ? ctx.branchesById.get(asset.branchId) : null;
    return b ? localize(b.name, ctx.locale) : '';
  }
  if (k === ASSIGNMENT_KINDS.EMPLOYEE) {
    const e = asset.assignedTo?.id ? ctx.employeesById.get(asset.assignedTo.id) : null;
    return e ? formatEmployeeName(e, ctx.locale) : '';
  }
  if (k === ASSIGNMENT_KINDS.DEPARTMENT) {
    return asset.assignedTo?.id ?? '';
  }
  return '';
}

/**
 * Build a workbook containing one sheet ("Assets") with rows 1+2 = headers,
 * row 3+ = data.
 *
 * @param {Array<import('@/domain/assets.js').Asset>} assets
 * @param {{
 *   categoriesById: Map<string, any>,
 *   statusesById: Map<string, any>,
 *   branchesById: Map<string, any>,
 *   employeesById: Map<string, any>,
 *   locale: string,
 *   labels?: Record<string, string>,
 * }} ctx
 *   `labels` is optional — when supplied, row 2 uses these strings;
 *   otherwise row 2 falls back to the COLUMN_LABEL_KEYS values verbatim.
 * @returns {XLSX.WorkBook}
 */
export function rowsToWorkbook(assets, ctx) {
  const headerRow1 = [...COLUMN_KEYS];
  const headerRow2 = COLUMN_KEYS.map((k) =>
    ctx?.labels?.[k] ?? COLUMN_LABEL_KEYS[k] ?? k
  );

  const aoa = [headerRow1, headerRow2];

  for (const a of assets ?? []) {
    const cat = ctx.categoriesById?.get(a.categoryId) ?? null;
    const names = nameForLocale(a, ctx.locale);
    const row = COLUMN_KEYS.map((k) => {
      switch (k) {
        case 'inventoryCode': return a.inventoryCode ?? '';
        case 'categoryId': return a.categoryId ?? '';
        case 'categoryName': return cat ? localize(cat.name, ctx.locale) : '';
        case 'nameRu': return names.ru;
        case 'nameEn': return names.en;
        case 'nameHy': return names.hy;
        case 'brand': return a.brand ?? '';
        case 'model': return a.model ?? '';
        case 'serialNumber': return a.serialNumber ?? '';
        case 'statusId': return a.statusId ?? '';
        case 'assignedToKind': return a.assignedTo?.kind ?? '';
        case 'assignedToId': return a.assignedTo?.id ?? '';
        case 'holderName': return holderName(a, ctx);
        case 'branchId': return a.branchId ?? '';
        case 'notes': return a.notes ?? '';
        case 'purchaseDate': return isoDateUTC(a.purchaseDate);
        case 'purchasePrice': return a.purchasePrice ?? '';
        case 'createdAt': return isoDateUTC(a.createdAt);
        default: return '';
      }
    });
    aoa.push(row);
  }

  const ws = XLSX.utils.aoa_to_sheet(aoa);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, buildExportSheetName());
  return wb;
}

export function workbookToBlob(wb) {
  const out = XLSX.write(wb, { bookType: 'xlsx', type: 'array' });
  return new Blob([out], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
  });
}

export function downloadFilename(date = new Date()) {
  return `assets_export_${isoDateUTC(date)}.xlsx`;
}

export function buildExportSheetName() { return 'Assets'; }
```

```js
/**
 * Pure-JS Excel import service for the assets workbook.
 *
 * Stays decoupled from React and Firestore: takes catalog snapshots in via
 * `ctx`, returns plain JS objects (`ValidationResult`). The dialog component
 * orchestrates side effects (reading the file, calling the repository).
 */

import * as XLSX from 'xlsx';
import { COLUMN_KEYS } from '@/lib/excel/columns.js';
import { ASSIGNMENT_KIND_LIST, ASSIGNMENT_KINDS } from '@/domain/assets.js';
import { DEFAULT_ASSET_STATUS_CODE } from '@/domain/assetStatuses.js';
import { localize } from '@/lib/localize.js';

const NON_ASCII_REGEX = /[^\x00-\x7F]/;

// ----- file → workbook ------------------------------------------------------

export async function workbookFromArrayBuffer(buf) {
  return XLSX.read(buf, { type: 'array' });
}

// ----- workbook → rows ------------------------------------------------------

/**
 * Read the first sheet (case-insensitive "Assets" preferred when present)
 * and return rows keyed by COLUMN_KEYS.
 */
export function workbookToRows(wb) {
  const sheetName = wb.SheetNames.find((n) => n.toLowerCase() === 'assets') ?? wb.SheetNames[0];
  if (!sheetName) return { headerKeys: [], rows: [], errors: ['errorImportEmptyFile'] };
  const ws = wb.Sheets[sheetName];
  const aoa = XLSX.utils.sheet_to_json(ws, { header: 1, raw: true, defval: '' });
  if (aoa.length === 0) return { headerKeys: [], rows: [], errors: ['errorImportEmptyFile'] };

  const headerRow1 = aoa[0].map((c) => String(c ?? '').trim());
  // Detect a label band on row 2: row 2 exists AND none of its cells equals a known column key.
  const knownKeys = new Set(COLUMN_KEYS);
  const looksLikeLabels =
    aoa[1] &&
    aoa[1].length > 0 &&
    aoa[1].every((c) => typeof c === 'string' && !knownKeys.has(String(c).trim()));
  const dataStart = looksLikeLabels ? 2 : 1;

  // Build a key → column index map. Unknown headers are ignored.
  const idx = new Map();
  for (let i = 0; i < headerRow1.length; i++) {
    if (knownKeys.has(headerRow1[i])) idx.set(headerRow1[i], i);
  }

  const rows = [];
  for (let r = dataStart; r < aoa.length; r++) {
    const raw = aoa[r];
    if (!Array.isArray(raw)) continue;
    // Skip wholly-empty rows.
    if (raw.every((c) => c == null || String(c).trim() === '')) continue;
    const obj = {};
    for (const key of COLUMN_KEYS) {
      const i = idx.get(key);
      obj[key] = i === undefined ? '' : raw[i] ?? '';
    }
    rows.push(obj);
  }
  return { headerKeys: headerRow1, rows, errors: [] };
}

// ----- per-row validation ---------------------------------------------------

function strOr(v) { return v == null ? '' : String(v).trim(); }
function asNumber(v) {
  if (typeof v === 'number') return Number.isFinite(v) ? v : null;
  const s = strOr(v);
  if (s === '') return null;
  const n = Number(s);
  return Number.isFinite(n) ? n : null;
}

/**
 * Parse a date cell.
 * Accepts: ISO yyyy-mm-dd string, full ISO datetime string, or Excel serial
 * number (XLSX cellDates: false leaves dates as numbers; we accept both).
 */
function parseDateCell(v) {
  if (v == null || v === '') return null;
  if (v instanceof Date) return Number.isNaN(v.valueOf()) ? null : v;
  if (typeof v === 'number') {
    // Excel serial date — epoch 1899-12-30 (the "1900 leap year bug" offset).
    const ms = Math.round((v - 25569) * 86400 * 1000);
    const d = new Date(ms);
    return Number.isNaN(d.valueOf()) ? null : d;
  }
  const s = String(v).trim();
  if (!s) return null;
  // Strict ISO yyyy-mm-dd
  const m = /^(\d{4})-(\d{2})-(\d{2})$/.exec(s);
  if (m) {
    const d = new Date(Date.UTC(Number(m[1]), Number(m[2]) - 1, Number(m[3])));
    return Number.isNaN(d.valueOf()) ? null : d;
  }
  // Looser: let Date parse; reject unrecognized.
  const d2 = new Date(s);
  return Number.isNaN(d2.valueOf()) ? null : d2;
}

function resolveCategory(rawRow, ctx) {
  const id = strOr(rawRow.categoryId);
  if (id) {
    const cat = ctx.categories.find((c) => c.categoryId === id);
    if (cat && cat.isActive !== false) return cat;
    return null;
  }
  const name = strOr(rawRow.categoryName);
  if (!name) return null;
  // Try matching against any locale of the name.
  const found = ctx.categories.filter((c) => {
    if (c.isActive === false) return false;
    return ['ru', 'en', 'hy'].some((l) => (c.name?.[l] ?? '').toLowerCase() === name.toLowerCase());
  });
  if (found.length === 1) return found[0];
  return null;
}

function resolveStatus(rawRow, ctx) {
  const id = strOr(rawRow.statusId);
  if (!id) return null; // caller treats null as "fallback to default".
  const found = ctx.statuses.find((s) => s.statusId === id && s.isActive !== false);
  return found ?? null;
}

function resolveBranch(id, ctx) {
  const s = strOr(id);
  if (!s) return null;
  const found = ctx.branches.find((b) => b.branchId === s && b.isActive !== false);
  return found ?? null;
}

function resolveEmployee(id, ctx) {
  const s = strOr(id);
  if (!s) return null;
  const found = ctx.employees.find((e) => e.employeeId === s && e.isActive !== false);
  return found ?? null;
}

/**
 * Validate one raw row from the workbook and produce a `ValidationResult`.
 *
 * @param {Record<string, unknown>} rawRow
 * @param {number} rowIndex
 * @param {{
 *   categories: any[], statuses: any[], branches: any[], employees: any[],
 *   existingInventoryCodes: Set<string>,
 * }} ctx
 */
export function validateRow(rawRow, rowIndex, ctx) {
  const errors = [];
  const warnings = [];

  // --- inventoryCode conflict (red short-circuit at the end so we still surface other reds first) ---
  const invCode = strOr(rawRow.inventoryCode);
  if (invCode && ctx.existingInventoryCodes?.has(invCode)) {
    errors.push({ rule: 'errorImportInventoryCodeConflict' });
  }

  // --- 1. category ---
  const cat = resolveCategory(rawRow, ctx);
  if (!cat) {
    errors.push({ rule: 'errorImportCategoryRequired' });
  }

  // --- 2. name ---
  const ru = strOr(rawRow.nameRu);
  const en = strOr(rawRow.nameEn);
  const hy = strOr(rawRow.nameHy);
  let name = null;
  if (cat) {
    if (cat.requiresMultilang) {
      const filled = [ru, en, hy].filter(Boolean);
      if (filled.length === 0) {
        errors.push({ rule: 'errorImportNameRequired' });
      } else {
        name = { ru, en, hy };
        if (filled.length < 3) warnings.push({ rule: 'warnImportNamePartialLocales' });
      }
    } else {
      // Single-lang: nameRu wins; fall back to first non-empty.
      const v = ru || en || hy;
      if (!v) errors.push({ rule: 'errorImportNameRequired' });
      else name = v;
    }
  }

  // --- 3. assignedToKind ---
  const kind = strOr(rawRow.assignedToKind).toLowerCase();
  if (!ASSIGNMENT_KIND_LIST.includes(kind)) {
    errors.push({ rule: 'errorImportAssignedKindRequired' });
  }

  // --- 4. warehouse + id present → warning ---
  let assignedToId = strOr(rawRow.assignedToId);
  if (kind === ASSIGNMENT_KINDS.WAREHOUSE && assignedToId) {
    warnings.push({ rule: 'warnImportWarehouseIdIgnored' });
    assignedToId = '';
  }

  // --- 5. holder existence ---
  if (kind === ASSIGNMENT_KINDS.EMPLOYEE) {
    if (!resolveEmployee(assignedToId, ctx)) errors.push({ rule: 'errorImportEmployeeUnknown' });
  } else if (kind === ASSIGNMENT_KINDS.BRANCH) {
    if (!resolveBranch(assignedToId, ctx)) errors.push({ rule: 'errorImportBranchUnknown' });
  } else if (kind === ASSIGNMENT_KINDS.DEPARTMENT) {
    if (!assignedToId) errors.push({ rule: 'errorImportDepartmentRequired' });
  }

  // --- 6. ASCII brand/model/serial ---
  for (const f of ['brand', 'model', 'serialNumber']) {
    const v = strOr(rawRow[f]);
    if (v && NON_ASCII_REGEX.test(v)) errors.push({ rule: 'errorImportAsciiOnly', field: f });
  }

  // --- 7. status (resolve with fallback) ---
  const rawStatusId = strOr(rawRow.statusId);
  let status = resolveStatus(rawRow, ctx);
  let statusId = status?.statusId ?? null;
  if (!status) {
    if (rawStatusId) {
      // unknown id → still fall back; warn.
      warnings.push({ rule: 'warnImportStatusFallback' });
    } else {
      // empty → fall back; warn.
      warnings.push({ rule: 'warnImportStatusFallback' });
    }
    statusId = DEFAULT_ASSET_STATUS_CODE;
    status = ctx.statuses.find((s) => s.statusId === DEFAULT_ASSET_STATUS_CODE) ?? null;
  }

  // --- 8. status × kind compatibility ---
  if (status && ASSIGNMENT_KIND_LIST.includes(kind)) {
    const wantAssignable = kind !== ASSIGNMENT_KINDS.WAREHOUSE;
    if (Boolean(status.isAssignable) !== wantAssignable) {
      errors.push({ rule: 'errorImportStatusKindMismatch' });
    }
  }

  // --- 9. branchId requirement ---
  let branchId = strOr(rawRow.branchId);
  if (kind === ASSIGNMENT_KINDS.WAREHOUSE || kind === ASSIGNMENT_KINDS.BRANCH) {
    if (!branchId) {
      errors.push({ rule: 'errorImportBranchIdRequired' });
    } else if (!resolveBranch(branchId, ctx)) {
      errors.push({ rule: 'errorImportBranchUnknown' });
    }
  } else {
    branchId = '';
  }

  // --- 10. purchaseDate ---
  const rawDate = rawRow.purchaseDate;
  let purchaseDate = null;
  if (rawDate !== '' && rawDate != null) {
    const parsed = parseDateCell(rawDate);
    if (!parsed) errors.push({ rule: 'errorImportPurchaseDate' });
    else purchaseDate = parsed;
  }

  // --- 11. purchasePrice ---
  let purchasePrice = null;
  if (rawRow.purchasePrice !== '' && rawRow.purchasePrice != null) {
    const n = asNumber(rawRow.purchasePrice);
    if (n == null || n < 0) errors.push({ rule: 'errorImportPurchasePrice' });
    else purchasePrice = n;
  }

  // --- aggregate ---
  if (errors.length > 0) {
    return { status: 'red', errors, warnings, normalized: null };
  }

  // Build normalized AssetInput consumable by `firestoreAssetRepository.create`.
  const normalized = {
    categoryId: cat.categoryId,
    name,
    brand: strOr(rawRow.brand) || null,
    model: strOr(rawRow.model) || null,
    serialNumber: strOr(rawRow.serialNumber) || null,
    statusId,
    assignedTo: kind === ASSIGNMENT_KINDS.WAREHOUSE
      ? { kind, id: null }
      : { kind, id: assignedToId },
    branchId: branchId || null,
    notes: strOr(rawRow.notes) || null,
    purchaseDate,
    purchasePrice,
    isActive: true,
  };

  return {
    status: warnings.length > 0 ? 'yellow' : 'green',
    errors: [],
    warnings,
    normalized,
  };
}

// ----- failure report -------------------------------------------------------

export function buildFailureReport(failures) {
  const aoa = [
    ['rowIndex', 'reasons', 'raw'],
    ...failures.map((f) => [
      f.rowIndex,
      (f.errors ?? []).map((e) => e.rule).join('; '),
      JSON.stringify(f.raw ?? {}),
    ]),
  ];
  const ws = XLSX.utils.aoa_to_sheet(aoa);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, 'Failures');
  return wb;
}
```
